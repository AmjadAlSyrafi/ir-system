Dataset 1

beir / quora / test  522k docs

Quora duplicate-question corpus. Short, natural-language questions on everyday topics. Binary relevance. All five models work well here — BM25 and TF-IDF are especially strong on exact keyword overlap.

bash — bm25 example
curl -s -X POST http://localhost:8000/search \
  -H "Content-Type: application/json" \
  -d '{
    "query": "What is the best way to learn programming?",
    "dataset": "dataset1",
    "model": "bm25",
    "top_k": 10,
    "use_refinement": true
  }'
Dataset 2

beir / hotpotqa / test  5.2M docs

HotpotQA multi-hop reasoning questions. Answers require combining information from two Wikipedia passages. Complex, compound queries — embedding and hybrid models shine here where BM25 struggles with implicit term overlap.

bash — hybrid_parallel example
curl -s -X POST http://localhost:8000/search \
  -H "Content-Type: application/json" \
  -d '{
    "query": "Were Scott Derrickson and Ed Wood of the same nationality?",
    "dataset": "dataset2",
    "model": "hybrid_parallel",
    "top_k": 10,
    "use_refinement": false
  }'
Reference

Models

Sparse models are always available immediately on startup. Dense and hybrid models finish loading in the background (~30–60s after first start).

ModelTypeLatencyDescription
tfidf Sparse < 5 ms Vector Space Model, cosine similarity over TF-IDF matrix.
bm25 Sparse < 5 ms Okapi BM25 — best sparse baseline. Tunable k1/b parameters.
embedding Dense ~50 ms msmarco-distilbert-base-v4 + FAISS flat inner-product index.
hybrid_serial Hybrid ~80 ms BM25 candidate retrieval → embedding re-rank pipeline.
hybrid_parallel Hybrid ~100 ms Reciprocal Rank Fusion across TF-IDF, BM25, and embedding.
